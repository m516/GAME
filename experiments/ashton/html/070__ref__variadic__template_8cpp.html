Detailed Description

Attention

At the time of writing, the part between <> is totally ignored: func<Args...>(Args... args) is interpreted as func(Args... args).

Beware that a function parameter with either a & or * operator, e.g. 'const Args&... args', requires \link and \ref to specify such parameter as verbatim, i.e. 'const Args&... args'. At the time of writing, the form func(const Args&...) will fail, unless the function parameter was declared just as 'const Args&...'.
